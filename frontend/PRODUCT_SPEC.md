## 1. 目的

### 解決する課題

東京通信大学（TOU）はオンライン完結型の通信大学であり、学生は**自分でスケジュールを立てて**学習を進める必要がある。しかし：

- 大学が提供する学事カレンダー（授業配信日・試験日・成績発表日）と、自分が立てた受講計画を**ひとつの場所で管理する手段がない**
- 複数科目の受講回（第1回〜第8回など）を日付に割り当てる作業が**手作業で煩雑**
- スケジュールを変更するたびに紙や Excel で書き直す**非効率**が生じている

### ツールの目的

> **大学の学事スケジュールと自分の受講計画を、ひとつのカレンダー上で一元管理・視覚化する**

---

## 2. 価値

### ユーザーへの主な提供価値

| 価値                       | 説明                                                                                                   |
| -------------------------- | ------------------------------------------------------------------------------------------------------ |
| **全体像の把握**           | 学事日程と自分の受講計画を同一カレンダーで重ねて見ることができ、試験前の詰め込みや締め切り漏れを防げる |
| **高速なスケジュール作成** | 複数科目の受講回をまとめて一括登録できる。手動登録の手間を大幅削減                                     |
| **直感的な変更操作**       | ドラッグ＆ドロップで予定を移動でき、「変更→確認→保存」のサイクルが短い                                 |
| **ミス・後悔のないUX**     | すべての変更操作に 15 秒間の「元に戻す」が付いており、誤操作を安心してリカバリーできる                 |
| **どこからでもアクセス**   | Web アプリかつアカウント管理済みのため、PC・スマホ・タブレットから同一データにアクセス可能             |

---

## 3. 体験

### 体験の設計思想

- **「登録した後にすぐ動かせる」**：ドラッグで予定を別の日に移動する感覚を中心に設計されている
- **「失敗を恐れない」**：すべての変更にアンドゥが付いており、取り消しがいつでもできる
- **「まとめて効率よく」**：科目・回数・日付範囲を指定するだけで大量のスケジュールを一括登録できる
- **「余計な操作をさせない」**：コンテキストメニュー（右クリック）、ポップオーバー、キーボードショートカットで画面遷移を最小化

### 主要ユーザーフロー

```
サインイン → カレンダーを確認 → スケジュール作成 → ドラッグで調整 → 完了
```

```
カレンダーを見る → 予定が変わった → ドラッグ or 編集ダイアログで変更 → アンドゥで確認
```

## 4. 機能仕様

### 4.1 スケジュールの種類

| 種類     | 表示名 | 説明                                                       | デフォルト色   |
| -------- | ------ | ---------------------------------------------------------- | -------------- |
| `master` | 学事   | 大学が定める学事カレンダー（授業配信・試験・成績発表など） | 青・赤など     |
| `custom` | 受講   | 学生が自分で登録する受講計画                               | 白・グレーなど |

### 4.2 スケジュール CRUD

#### 作成

- **単体作成**：名前・種類（学事/受講）・開始日・終了日・色を指定して1件登録
- **一括作成（範囲）**：「第○回〜第○回」という連番名称で複数件登録（例：第1回〜第8回で8件）
- **一括作成（テンプレート）**：登録済み科目を使い「科目名 第○回」形式で登録
- **一括作成（プレフィックスなし）**：科目名のみで複数件登録

#### 編集

- ダイアログから名前・日付・色・種類を変更
- ドラッグ＆ドロップで日付を変更（セル間移動）
- 同一セル内でのドラッグで表示順序を変更

#### 削除

- 単体削除：コンテキストメニューまたは編集ダイアログから
- 一括削除：選択モードで複数選択後に Delete/Backspace キー または ボタン

#### アンドゥ（元に戻す）

- 作成・編集・削除・移動・**色変更**のすべての操作後にトーストが表示
- トースト上の「元に戻す」ボタンを押すと **15 秒以内** に操作前の状態に戻せる

### 4.3 ドラッグ＆ドロップ

| モード                 | 説明                                                                                         |
| ---------------------- | -------------------------------------------------------------------------------------------- |
| **シングルドラッグ**   | 1件のスケジュールを別の日付セルへ移動                                                        |
| **バルクドラッグ**     | 複数選択されたスケジュールを同時に移動。主アイテムを基準に他のアイテムの日付差分が計算される |
| **同一セル内ドラッグ** | 同じ日付セル内でのドラッグにより表示順序（order）を変更                                      |

**視覚フィードバック**：

- ドラッグ中のアイテムはオーバーレイ表示（名前と日付範囲を表示）、わずかに回転（`rotate-1`）＋拡大（`scale-105`）する
- 元アイテムは半透明（`opacity-30`）になり、ドラッグ中であることを示す
- バルクドラッグ時はバッジで件数を表示（例：「3件移動中」）
- ドロップ先のセルが青くハイライトされ、種別ラベル（「学事」「受講」）が青バッジで表示される
- ドロップ後はアニメーション（220ms、cubic-bezier）で着地する

**ドラッグの挙動詳細**：

- ドラッグ開始時に選択状態が全クリアされる
- 同一セル内ドラッグ：dragOver でライブに並び替え（API 呼び出しなし）、dragEnd 時にのみ API 保存
- セル間ドラッグ：dragOver で楽観的 UI 更新、dragEnd で API 保存
- バルクドラッグ：主アイテムのみ dragOver で移動、他アイテムは dragEnd で日付差分を計算して一括移動
- ドロップ先がない場合（キャンセル）：dragStart 時のスナップショットで UI を復元

### 4.4 複数選択モード

- 右クリック（PC）または長押し（モバイル）→ コンテキストメニュー →「選択」で選択モード開始
- **PC**：Ctrl+クリック（Mac は Cmd+クリック）で選択トグル、Shift+クリックで範囲選択（同一日付内）、選択モード中はクリックでもトグル
- **モバイル**：長押しで選択モード開始またはトグル、タップで選択トグル（選択モード中）
- 別の日付のスケジュールを選択すると、選択がその1件にリセットされる
- 選択モード中に最後の1件を解除すると自動的に選択モードが終了する
- Escape キーで選択解除・選択モード終了
- 選択中は画面上部に SelectionModeBar を表示

### 4.5 カレンダーナビゲーション

- 前月・当月（今日）・翌月ボタンで移動
- 無限スクロール：上下スクロールで過去・未来の月を追加表示
- 今日の日付がセル上でハイライト表示される

### 4.6 科目管理

- 科目を登録しておくと、受講スケジュール作成ダイアログでチップとして表示される
- チップをクリックすると名前と色が自動入力される（単体作成・一括作成どちらでも使用可能）
- 科目には名前と色を設定
- 設定画面（`/settings`）から追加・削除できるほか、作成ダイアログ内からも直接追加可能

### 4.7 マスタースケジュールテンプレート

学事スケジュール作成時に名前と色をワンタッチで入力できるテンプレートチップが用意されている：

| 第3回 授業配信    | 黄  |
| 第4回 授業配信    | 黄  |
| 第5回 授業配信    | 黄  |
| 第6回 授業配信    | 黄  |

### 5.3 日付セルの構造

各日付セルには2行のスケジュールエリアがある：

```
┌────────────────┐
│  1 (日付)      │
│  [学事スケジュール] ← master 行
│  [受講スケジュール] ← custom 行
└────────────────┘
```

- 複数日にまたがるスケジュールは CSS Grid の colspan で横に伸びて表示
- 土曜日は青文字、日曜日は赤文字で表示
- 今日の日付セルは背景色でハイライト

### 5.4 スケジュールアイテムの操作

| 操作                               | 動作                                                                       |
| ---------------------------------- | -------------------------------------------------------------------------- |
| クリック（PC）                     | InfoCard ポップオーバーを表示                                              |
| タップ（モバイル）                 | InfoCard ポップオーバーを表示                                              |
| Enter / Space（フォーカス中）      | InfoCard ポップオーバーを表示                                              |
| Ctrl+クリック / Cmd+クリック（PC） | 選択トグル                                                                 |
| Shift+クリック（PC）               | 範囲選択（同一日付内）                                                     |
| 右クリック（PC）                   | コンテキストメニューを表示                                                 |
| 長押し 500ms（モバイル）           | 選択モード開始 または 選択トグル（選択モード中）＋バイブレーション（30ms） |
| 選択モード中タップ                 | 選択トグル                                                                 |
| ドラッグ                           | スケジュールを別日付に移動（開始時にバイブレーション 10ms）                |


- 「選択」を押すと選択モードに入り、そのアイテムが最初の選択状態になる

### 5.7 スケジュール作成ダイアログ

ダイアログタイトル：**スケジュールを追加**

単一ダイアログ内にコンテンツを条件表示する構成（多段階ではない）：

```
┌──────────────────────────────────────────────┐
│ スケジュールを追加                             │
│                                              │
│ [受講] [学事]  [●色]                         │
│                                              │
│ 名前: [_______________________________]      │
│                                              │
│ ＜学事選択時のみ＞                             │
│ テンプレート: [履修登録][第1回 授業配信]...    │
│                                              │
│ ＜受講選択時のみ＞                             │
│ 科目: [数学][英語][+] ← クリックで名前・色を入力 │
│                                              │
│ □ 第[1]回 から 第[8]回 で一括作成する          │
│ □ 第[1]回 で全テンプレート分を一括作成する      │
│ □ 全テンプレート分を一括作成する                │
│                                              │
│ 開始日: [__________]                         │
│ 終了日: [__________]                         │
│                                              │
│                    [作成]  [キャンセル]       │
└──────────────────────────────────────────────┘
```

- **学事テンプレートチップ**：クリックすると名前と色が自動入力される
- **受講科目チップ**：クリックすると名前と色が自動入力される（単体作成にも一括作成にも使える）。`[+]` から作成ダイアログ内で科目を直接追加可能
- 一括作成オプションは同時に1つしか有効にできない（範囲・テンプレート・プレフィックスなしは排他）
- テンプレート一括時は登録済み科目全件分を一度に作成
- 開始日・終了日は blur 時に自動補正される（終了日 < 開始日 の場合、もう一方を合わせる）

### 5.8 SelectionModeBar

複数選択モード中に画面上部（ヘッダー直下）に表示されるバー：

```
┌────────────────────────────────────────────────────────────┐
│ 3件選択中                               [キャンセル] [削除] │
│ Ctrl+クリック で追加選択 / Shift+クリック で範囲選択         │
└────────────────────────────────────────────────────────────┘
```

- PC では「Ctrl+クリック で追加選択 / Shift+クリック で範囲選択」を表示
- モバイルでは「長押しで選択モード開始 / タップで追加・解除」を表示
- 「キャンセル」で選択モード終了、「削除」は0件選択時は非活性

### 5.9 キーボードショートカット

| キー                   | 動作                                             |
| ---------------------- | ------------------------------------------------ |
| `Enter` / `Space`      | フォーカス中のスケジュールの InfoCard を開く     |
| `Delete` / `Backspace` | 選択中のスケジュールを削除（グローバルリスナー） |
| `Escape`               | 選択モードを終了 / ダイアログを閉じる            |

### 5.10 トースト通知

操作後に画面下部に表示：

```
┌───────────────────────────────────┐
│ スケジュールを作成しました  [元に戻す] │
└───────────────────────────────────┘
```

- 表示時間：15 秒間
- 「元に戻す」を押すと操作を取り消し
- セッション切れ時は専用トーストを表示：「セッションが切れました。再ログインしてください。」→ サインインページへ自動遷移

### 5.11 レスポンシブ対応

| デバイス   | 対応内容                                                                                                            |
| ---------- | ------------------------------------------------------------------------------------------------------------------- |
| PC         | 全機能をフル活用。週ビューを横に広く表示                                                                            |
| タブレット | カレンダーレイアウトを調整して表示                                                                                  |
| スマホ     | 長押し（500ms）でドラッグ or 選択モード開始。タップでメニュー・選択操作。触覚フィードバック（バイブレーション）あり |

## 6. データモデル

### 6.1 Schedule（スケジュール）

```typescript
type Schedule = {
  id: string; // UUID
  name: string; // スケジュール名（例: "数学 第3回"）
  startDate: Date; // 開始日
  endDate: Date; // 終了日
  color: string; // カラーキー（例: "blue", "outline_orange"）
  type: 'master' | 'custom'; // 種類
  order: number; // 同一日付セル内の表示順
};
```

### 6.2 ScheduleDateItem（日付別スケジュール）

```typescript
type ScheduleDateItem = {
  date: string; // 'yyyy-MM-dd' 形式
  type: 'master' | 'custom';
  schedules: Schedule[]; // その日付・種類のスケジュール一覧
};
```

### 6.3 Subject（科目）

```typescript
type Subject = {
  id: string;
  userID: string;
  name: string; // 科目名（例: "数学"）
  color: string; // デフォルト色
  createdAt: string;
  updatedAt: string;
};
```

### 6.4 AuthUser（認証ユーザー）

```typescript
type AuthUser = {
  id: string;
  email: string;
  name: string;
  session_token: string; // snake_case のまま localStorage に保存
};
```

localStorage のキー名：`'auth-user'`（JSON 文字列）。有効期限なし（明示的なサインアウトまたはセッション切れで削除）。

## 9. 利用手順

### 9.1 アカウント作成

1. `/signup` ページを開く
2. メールアドレスを入力して「登録」ボタンを押す
3. 届いたメールのリンクから `/password/set` ページへアクセス
4. パスワードを設定する
5. `/signin` からサインインする

### 9.2 初めてのスケジュール作成

1. **ヘッダーの「作成」ボタン**（＋アイコン付き青色ボタン）を押す
2. ダイアログが開く。デフォルトは種類「**受講**」が選択された状態
3. 種類を必要に応じて変更する：
   - **学事**：大学の公式日程（授業配信日・試験日など）
   - **受講**：自分が受ける授業の予定
4. 名前を入力する（学事の場合はテンプレートチップをクリックすると自動入力）
5. 開始日・終了日を**選ぶ**
6. 色を選ぶ
7. 「作成」ボタンを押す → カレンダーに表示される

### 9.3 複数科目をまとめて登録する（一括作成）

1. 事前準備：設定画面（`/settings`）で科目を登録する
   - 科目名（例：「数学」「英語」）と色を設定
2. 「作成」ボタン → 種類「受講」を選択
3. ダイアログ内に一括作成オプション（チェックボックス）が表示される
   - **第○回 から 第○回 で一括作成する**：連番名称で登録（例：「第1回 から 第8回」を指定すると「第1回 科目名」…「第8回 科目名」を一括登録）
   - **第○回 で全テンプレート分を一括作成する**：登録済み科目それぞれを指定回数で一括登録（例：「第3回 数学」「第3回 英語」など）
   - **全テンプレート分を一括作成する**：登録済み科目の名前そのままで一括登録
4. 開始日・終了日を入力する
5. 「**作成**」ボタンを押す → 複数件がまとめて登録される

### 9.4 スケジュールを別の日に移動する

**ドラッグ&ドロップで移動**：

1. 移動したいスケジュールをドラッグ（クリックしたまま移動）
2. 移動先の日付セルまでドラッグ
3. セルが青くハイライトされたら放す

**編集ダイアログで移動**：

1. 移動したいスケジュールをクリック → InfoCard が開く
2. 鉛筆アイコン（編集）を押す
3. 日付を変更して「保存」

### 9.5 複数のスケジュールをまとめて移動する

1. 任意のスケジュールを右クリック（PC）または長押し（モバイル）→「選択」
2. 同じ日付の他のスケジュールをクリック（PC は Ctrl+クリック）して追加選択
   - **別の日付のスケジュールをクリックすると選択がリセットされその1件のみになる**
3. 選択したスケジュールのどれかをドラッグ
4. 移動先の日付にドロップ → 選択された全スケジュールが日付差分を保ったまま同時に移動

### 9.6 スケジュールを削除する

**単体削除**：

1. 削除したいスケジュールを右クリック → 「削除」
2. 確認ダイアログで「削除」ボタンを押す

**複数削除**：

1. 選択モードで複数選択
2. `Delete` / `Backspace` キーを押す または SelectionModeBar の「削除」ボタン
3. 確認ダイアログで「削除」ボタンを押す

### 9.7 操作を元に戻す（アンドゥ）

- 作成・編集・削除・移動・**色変更**の後にトースト通知が表示される
- **15 秒以内** にトーストの「元に戻す」ボタンを押すと操作前の状態に戻る
- 15 秒経過するとトーストが消え、アンドゥできなくなる

### 9.8 スケジュールの表示順を変える

1. 同じ日付セル内で、スケジュールをドラッグして上下に動かす
2. 放した位置に並び順が変更される

### 9.9 月の移動

- **「←」ボタン**：前の月に移動
- **「今月」ボタン**：今月（今日が含まれる月）に戻る
- **「→」ボタン**：翌月に移動
- **上下スクロール**：無限スクロールで過去・未来の月へ自動追加

### 9.10 アカウント設定

1. ヘッダー右上のアカウントアイコンをクリック
2. ドロップダウンメニューが表示される（項目：「**設定**」「**サインアウト**」）
3. 「設定」をクリックして `/settings` ページへ移動
4. 設定ページでは以下が可能：
   - **名前の変更**
   - **メールアドレスの変更**（変更メールを送信）
   - **科目の追加・削除**
   - **アカウント削除**

## 11. 制約事項

操作や機能に関する重要な制限・注意事項をまとめる。

| カテゴリ     | 制約                                                                                                           |
| ------------ | -------------------------------------------------------------------------------------------------------------- |
| 複数選択     | 選択は同一日付セル内のみ有効。別の日付を選ぶと選択がリセットされその1件だけになる                              |
| 複数選択     | 選択モード中に最後の1件を解除すると選択モードが自動終了する                                                    |
| 複数選択     | ドラッグ開始時に選択状態が全クリアされる（バルクドラッグ中は除く）                                             |
| 一括作成     | 一括作成オプション（範囲・テンプレート・プレフィックスなし）は同時に1つしか有効にできない                      |
| 一括作成     | テンプレート一括作成は登録済み科目が0件の場合、何も作成されない                                                |
| アンドゥ     | 「元に戻す」はトースト表示後 15 秒以内のみ有効                                                                 |
| アンドゥ     | 次の操作を行うとそれ以前のアンドゥは上書きされる                                                               |
| 日付入力     | 終了日 < 開始日の状態でフォーカスを外すと自動補正される                                                        |
| セッション   | セッショントークンは localStorage に有効期限なしで保存。401 エラー時に自動削除＋サインインページへリダイレクト |
| バックエンド | バックエンドサーバー（デフォルト `http://localhost:3000`）が起動していないとすべての読み書き操作が失敗する     |

## 13. 類似ツールとの差別化

Notion やスプレッドシートのようにカスタマイズ性の高いツールでも、工夫すれば近い機能を再現できる場合がある。しかし**構築コスト（スキーマ設計・テンプレート整備・レイアウト作成）が高く、かつ汎用基盤の制約により最適化にも限界がある**。本ツールは「TOU 学生が受講スケジュールを管理する」という用途に最初から特化しているため、余計な設定なしに最適化された体験をそのまま提供できる。

**本ツール最大の固有価値**は、学事スケジュールと受講スケジュールを同一週グリッド上で上下に固定分離して表示することにある。この2トラック構造は汎用カレンダーでは再現できず、TOU 向けに特化した一括作成・テンプレートと合わせて、他ツールでは数十分かかる学期セットアップを数分で完了できる。

### 13.1 比較サマリー

✅ 完全対応 ／ △ 部分対応・手作業要 ／ ❌ 非対応

| 機能・特性                                  | 本ツール | Google カレンダー |     Notion     | Excel / スプレッドシート | 紙の手帳 |
| ------------------------------------------- | :------: | :---------------: | :------------: | :----------------------: | :------: |
| 学事と受講を上下固定分離して表示            |    ✅    |  △ 混在表示のみ   | △ 混在表示のみ |         △ 手作業         | △ 手書き |
| TOU 学事テンプレート（13種）の1クリック入力 |    ✅    |        ❌         |  △ 初期整備要  |            ❌            |    ❌    |
| 受講回の一括作成（第1回〜第8回 × 複数科目） |    ✅    |        ❌         |       ❌       |         △ 手作業         |    ❌    |
| ドラッグ&ドロップによる日程変更             |    ✅    |        ✅         |   △ 制限あり   |            ❌            |    ❌    |
| 複数アイテムの同時ドラッグ移動              |    ✅    |        ❌         |       ✅       |            ❌            |    ❌    |
| 全操作のアンドゥ                            |    ✅    |        ✅         |       ✅       |            ✅            |    ❌    |
| 週グリッドでの期間スケジュール可視化        |    ✅    |        ✅         |       △        |            ❌            |    △     |
| セットアップ不要で即使用可能                |    ✅    |        ✅         |  ❌ DB設計要   |    ❌ テンプレ作成要     |    ✅    |
| スマホからの同期アクセス                    |    ✅    |        ✅         |       ✅       |            ✅            |    ❌    |

### 13.2 ツール別の詳細比較

#### Google カレンダー

**できること**：汎用的な予定管理、招待・共有、スマホ同期、ドラッグで日程変更

**本ツールとの主な差**：

- 複数のカレンダーを色分けして同時表示することで学事と受講を並べて見ることはできるが、日付セル内で**学事を上・受講を下に常に分離して配置する**レイアウトは実現できない。混在して並ぶため、種類をひとめで区別しにくい
- 「第1回〜第8回」という授業回の**連番一括作成**がない。8科目 × 8回 = 64件を手動で1件ずつ作成する必要がある
- TOU 固有の学事スケジュール名（「単位認定試験」「成績評価」など）のテンプレートがなく、毎回手入力が必要
- 複数予定の**同時ドラッグ移動**がない

#### Notion（カレンダービュー）

**できること**：データベース + カレンダービュー、柔軟なプロパティ設計、チームでの共同編集、複数アイテムの同時移動

**本ツールとの主な差**：

- 学事スケジュールと受講スケジュールを同時表示できるが、日付セル内で**学事を上・受講を下に常に固定分離して表示する**レイアウトは実現できない。両者が混在して並ぶため種類を即座に区別しにくい
- 使い始めるには**データベースのスキーマ設計**が必要で、学習コストが高い。本ツールは開いた瞬間から使える
- テンプレート機能で学事スケジュールの雛形は作れるが、TOU の学事サイクルに合わせた13種を**初期に自前整備する**作業が発生する。受講回の連番一括作成（第1回〜第8回）も手作業が必要

#### Excel / Google スプレッドシート

**できること**：自由なレイアウト、計算式、ファイルとして保存・共有

**本ツールとの主な差**：

- カレンダー形式のビューを作るには**自分でシートを設計**する必要があり、維持コストが高い
- 日程変更は**セルの切り貼り**であり、ドラッグで直感的に動かせない
- スケジュール間の期間（開始日〜終了日）を**横スパンで視覚化**することが難しい
- Google スプレッドシートはスマホでの閲覧・編集は実用的だが、カレンダーレイアウトを自前で設計・維持する**運用コストは変わらない**
- ローカルの Excel ファイルは複数デバイスからの最新版アクセスに手間がかかる

#### 紙の手帳

**できること**：手書きの自由度、電源不要、直感的な書き込み

**本ツールとの主な差**：

- 日程変更のたびに書き直しが発生し、**変更コストが高い**
- 複数科目の授業回をすべて手書きするのは非常に手間がかかる
- スマホや PC からアクセスできず、持ち歩かないと確認できない
- 全体の俯瞰（どの週に何件集中しているか）が把握しにくい

### 13.3 本ツールが際立つユースケース

1. **学期はじめの一括セットアップ**：履修登録・授業配信スケジュール・試験日を一度にカレンダーに流し込む作業。他ツールでは数十件の手入力が必要なところを、テンプレートと一括作成で数分で完了する

2. **試験直前の計画見直し**：試験日や授業配信回が確定したあと、自分の受講ペースを調整する作業。ドラッグ&ドロップと複数同時移動で素早く並び替えられる

3. **学事スケジュールとの照合**：受講計画が試験日・成績発表日と重なっていないかを一目で確認できる。他ツールでは2つの情報源を行き来する必要がある

4. **特定科目の日程を丸ごとシフトする**：ある科目の受講ペースを全体的に後ろ倒しにしたい場合、その科目の受講回をまとめて選択し、ドラッグ1回で日付差分を保ったまま全件移動できる。他ツールでは1件ずつ手作業になる

5. **学事スケジュール変更後の即時対応**：大学側の授業配信日や試験日が変更になった際、受講計画との衝突箇所を週グリッド上で即座に見つけ、ドラッグで調整できる。Excel や紙では全体の再計画が必要になる
